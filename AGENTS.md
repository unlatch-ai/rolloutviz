# Agent instructions

RLViz is a local-first, source-read-only rollout viewer. Do not add network
calls, telemetry, hosted dependencies, source mutation, or execution of
recorded tools without an explicit product decision.

## Read before changing

- All behavior: `docs/product-spec.md` and `docs/architecture.md`
- UI: `docs/ui-information-architecture.md` and `docs/design-system.md`
- Canonical records: `docs/data-model.md` and the relevant schema/protocol
- Adapters or analyzers: `docs/plugin-model.md` and the relevant protocol
- Onboarding or CLI: `docs/onboarding.md` and `docs/supported-formats.md`
- Tests or fixtures: `docs/testing.md`
- Releases: `docs/releasing.md`

## Change rules

- Prefer small vertical changes that leave a usable CLI-to-browser path.
- Do not create speculative abstractions for hosted or enterprise features.
- Keep `docs/architecture.md` aligned with the actual repository and runtime.
- Protocol changes update Go and TypeScript types, schemas, fixtures,
  documentation, index/API mappings, and conformance tests together.
- Visible UI changes use design tokens, preserve keyboard access, include a
  browser-level test, and are verified in the real browser at a fixed viewport.
- Do not solve format-specific semantics with format-specific UI branches;
  improve the canonical vocabulary or use a declared presentation hint.
- Do not allow arbitrary plugin JavaScript or CSS in the core viewer.
- Keep test fixtures synthetic and free of customer traces, secrets, and
  proprietary model output.
- `web/dist` is generated by `npm --prefix web run build` and must match `web/src`
  in commits that change the embedded viewer.

## Verification

Run `make check` before declaring implementation work complete. Add focused
tests for the changed surface before the full check. Follow the additional
quality gates in `docs/testing.md` as they are implemented.
